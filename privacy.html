Privacy Policy

Last Updated: June 2026

This Privacy Policy explains how Kselis and our integrated utilities (including Kangaroo / KangaRu) handle, process, and secure information. As an open-source initiative aimed at democratizing education and workspace efficiency, we prioritize a strict non-custodial data model.

1. Data Collection and Processing

Our tools operate locally within your designated server environment. We only capture and process information explicitly required to execute core features:

  • Organization Profile Data: Including Discord Guild IDs, Owner User IDs, and related descriptive information necessary for bot registration.
  • Branding Assets: Image URLs for custom corporate logos or server icons provided during setup.
  • Operational Schedules: Calendar events, task descriptions, and execution timestamps logged by workspace members.
  • User Interactions: Real-time processing of Slash Commands and Form Modals to generate immediate software responses.

2. Data Storage and Sovereignty

Unlike standard commercial applications, our architecture does not transmit data to external, third-party centralized databases.

  • Local Storage: All profile data and workspace logs are written directly to local physical state files (e.g., company.json) residing on your host machine.
  • Data Control: Because storage is completely local, your server administrators have absolute sovereignty over the data and can delete, modify, or back up files manually at any time.

3. Data Retention and Deletion

Core profile data, timesheet structures, and calendar events are retained indefinitely on your host machine to ensure the continuous operation of workspace commands. If an organization wishes to completely remove all data, an administrator simply needs to delete the local JSON database files from the host server.

4. Third-Party Sharing

We do not sell, lease, trade, or distribute organizational data to any third-party marketing services or data brokers. Data transmitted through official APIs (such as Discord or Google Cloud) to facilitate text and embed message rendering is subject to the respective provider's Developer Terms of Service.

Note for Server Administrators: To safeguard information, ensure proper firewall and SSH access controls are configured on your hosting Virtual Machine instance to prevent unauthorized reading of your local database assets.
